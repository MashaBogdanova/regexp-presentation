What REGEXP is?

  • a powerful way to search and replace in text
  • is used in main programing languages
  • in compilers and interpreters for lexical analysis and parsing of source code
  • in text editors for text searching and replacing

REGEXP is used for

  • search and replace text
  • data validation
  • data filtering
  • formatting changes
  • information extraction

REGEXP in Java Script

Object REGEXP

Constructor function


							const myRegex = new RegExp(pattern, flags)

Literal notation


							const myRegexp = /pattern/flags

Patterns

the text of REGEXP


							const myRegexp = /text to find/

Flags

  • g - all matches vs. the first match
  • i - case-insensitive search
  • m - multiline mode
  • s - dotAll mode
  • u - unicode
  • y - sticky

Methods

integrated to string methods

str.match(regexp)

finds matches of regexp in a string


    let str = "We will, we will rock you"

    str.match(/we/) // [we]
    str.match(/we/i) // [We]
    str.match(/we/gi) // [We, we]
    str.matchAll(/we/gi) // We, we
    str.match(/WE/) // null

str.search(regexp)

returns the position of the first match or -1 if none found


    let str = "We will, we will rock you"
    str.search( /will/i ) // 3

str.replace(regexp, replacement)

replaces matches of regexp in a string with replacement


    let str = "We will, we will rock you"

    str.replace(/we/i, "I") // "I will, we will rock you"
    str.replace(/we/gi, "I") // "I will, I will rock you"
    str.replaceAll(/we/i, "I") // "I will, I will rock you"

regexp.test(str)

looks for at least one match, if found, returns true, otherwise false


    let str = "We will, we will rock you"

    str.test(/we/) // true

str.split(regexp|substr, limit)

splits the string using the regexp as a delimiter


    '12, 34, 56'.split(/,\s*/) // ['12', '34', '56']

Useful features

  • Character classes
  • Anchors
  • Sets and ranges
  • Quantifiers
  • Capturing groups

Character classes


   let time = "Exact time: 12:34"
   let regexp = /\d\d:\d\d/
   time.match(regexp) // 12:34
  • \d - digit [0-9]
  • \D - not digit
  • \s - space [\t\n\v\f\r ]
  • \S - not space
  • \w - letter, digit or underscore [a-zA-Z0-9_]
  • \W - not letter, not digit, not underscore
  • \. - any character except a newline

Anchors

  • ^ - matches at the beginning of the text
  • $ – matches at the end

   let str = `We will,
              we will rock you`
   /^We/.test(str) // true
   /you$/.test(str) // true
   /^we/gim.test(str) // true

Sets and ranges

searching for any character among given in [ ] or except the given ones


    'Mop top'.match(/[tm]op/gi) // 'Mop', 'top'
    'Exception 0xAF'.match(/x[0-9A-F][0-9A-F]/g) // xAF
    'mashabogdanoff@gmail.com'.match(/[^\d\A-Z]/gi) // @ и .

Quantifiers

marking how many symbols we need

  • + = {1,}
  • ? = {0,1}
  • * = {0,}

   "I'm 12345 years old".match(/\d{5}/) // 12345
   "I'm not 12, but 1234 years old".match(/\d{3,5}/) // 1234
   '+7(903)-123-45-67'.match(/\d{1,}/g) // 7,903,123,45,67

Alternation


                    let regexp = /html|php|css|java(script)?/gi
                    'First HTML appeared, then CSS, then JavaScript'.match(regexp) // HTML, CSS, JavaScript

Capturing groups

if we put a quantifier after the parentheses, it applies to the parentheses as a whole


                    let regexp = /[-.\w]+@([\w-]+\.)+[\w-]+/g
                    'my@mail.com @ his@site.com.uk'.match(regexp) // my@mail.com, his@site.com.uk

Pros and cons

  • + allows to find patterns
  • + powerful and flexible tool
  • + performance improvement
  • - difficult to learn and understand
  • - error-prone
